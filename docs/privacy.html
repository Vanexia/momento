Privacy Policy

Effective: 27 May 2026

Overview

Momento is a desktop application that runs locally on your Windows machine. It records gameplay, stores recordings on your local drive, and includes an optional feature to upload selected clips to YouTube on your behalf.

This policy explains what data Momento accesses, where it's stored, and what we do (and don't) do with it.

Short version: Momento collects no personal data and sends nothing over the network unless you explicitly click "Upload to YouTube" on a clip. There is no telemetry, no analytics, no account system, no advertising, and no third-party data sharing. Your recordings stay on your machine.

What Momento stores locally

Momento writes the following to your machine under %APPDATA%\Momento\ and the recordings folder you select in Settings:

  • Recordings (.mkv files) and exported clips (.mp4) — saved to the folder you choose. Momento never moves or transmits these files unless you explicitly export or upload them.
  • Bookmarks (.bookmarks.json sidecar files) — timecodes you marked during gameplay.
  • Thumbnails (.thumb.jpg) — extracted frames used by the in-app editor.
  • Settings (config.json) — your preferences (capture quality, hotkey, output folder, etc.).
  • Logs (logs/momento.log) — diagnostic messages, written only to disk, never transmitted.

Momento does not collect or transmit your machine specs, IP address, usage patterns, installed games, or any other system or behavioural data.

YouTube upload feature

If you choose to connect your YouTube account and upload a clip, Momento interacts with Google's APIs as described below. This feature is entirely optional and never runs without explicit user action.

Sign-in

Connection happens through Google's standard OAuth 2.0 desktop flow. When you click "Connect YouTube account," Momento opens your system's default browser and directs it to accounts.google.com. You sign in to Google on Google's site, review the requested permissions, and consent.

Momento never sees, requests, transmits, or stores your Google password. Authentication happens entirely between your browser and Google's servers.

What Google data Momento accesses

Momento requests two YouTube API scopes:

  • youtube.upload — permission to upload videos to your YouTube channel. Used solely to upload clips that you have explicitly selected and confirmed for upload through Momento's UI. Never used for any other purpose. Never used to upload content automatically or in the background.
  • youtube.readonly — permission to read your channel information. Used solely to display your channel name in Momento's Settings screen (so you can see which account you are connected as). The only API call made with this scope is a single channels.list(mine=true) request after sign-in, which retrieves your channel's name and ID. No other channel data, video data, comment data, or analytics data is retrieved or stored.

What Momento stores from Google

After successful sign-in, Momento stores the following locally on your machine:

  • OAuth refresh token — at %APPDATA%\Momento\youtube_token.dat. This token allows Momento to upload on your behalf without re-prompting you to sign in every time. It is encrypted using Windows Data Protection API (DPAPI), which binds the encrypted blob to your Windows user account. The encrypted file cannot be decrypted by another Windows user account on the same machine or by anyone on a different machine.
  • Connected channel name and ID — cached in config.json only so Momento's Settings can display "Signed in as: [your channel]" without making a network call on every Settings open.

No other Google-sourced data is stored.

Where uploaded videos go

When you click "Upload to YouTube" on a clip, Momento sends the video file directly to YouTube's servers via the official YouTube Data API v3. The video, title, description, tags, category, and privacy setting you chose in the upload dialog are uploaded. Momento does not send these to any other party. Once on YouTube, the video is governed by your YouTube account's settings and Google's own terms.

Compliance with Google API Services User Data Policy

Momento's use of information received from Google APIs adheres to the Google API Services User Data Policy, including the Limited Use requirements. Specifically:

  • Google user data is used only to provide the user-facing YouTube upload feature.
  • Google user data is not transferred to others except as needed to provide that feature (i.e., transferred to YouTube as the upload destination, on user instruction).
  • Google user data is not used for advertising, profiling, or any unrelated purpose.
  • Google user data is not read or stored by humans (the developer), except where the user explicitly opts to debug an issue and shares logs.

How to disconnect or revoke access

You can disconnect at any time:

  • From within Momento: Settings → YouTube → Disconnect. This deletes the local youtube_token.dat file immediately.
  • From your Google account: Visit myaccount.google.com/permissions, find "Momento," and click Remove access. This revokes Momento's ability to use any previously-issued tokens, even if the local token file still exists.

Either action stops Momento from making any further YouTube API calls on your behalf.

Children

Momento is not directed at children under 13. The app contains no advertising and collects no personal information regardless of user age. The YouTube upload feature inherits YouTube's own age policies.

Changes to this policy

If this policy changes meaningfully, the updated version will be posted here with a revised "Effective" date. Significant changes that affect how user data is handled will be accompanied by an in-app notice on the next launch after the change.
